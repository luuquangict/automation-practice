2. Sử dụng ID

Theo quy ước thì ID luôn là duy nhất trong 1 tài liệu HTML. Về cơ bản khi sử dụng ID thì sẽ lấy được Element ngay

Tuy nhiên, vãn sẽ có trường hợp đặc biệt khi vì một lý do nào đó, ông DEV ông vô tình hoặc cố ý làm cho ID trên tài liệu HTML bị trùng. Nên cần chú ý

//button[@id='btnExample1']

3. Sử dụng class

Dùng tới class của button khi button có 1 chứa class đặc biệt

//button[@class='btn btn-primary btn5']

Tuy nhiên ở trường hợp trên ta đang sử dụng dấu = là so sánh tuyệt đối, tức là bắt buộc class phải là btn btn-primary btn5

Khi bị thiếu một class hoặc thừa một class (ví dụ bị đổi thành btn btn-success btn5) thì ngay lập tức xpath bị sai ngay

Mà class thì là cái rất dễ bị thay đổi, vì vậy nên hạn chế sài class hơn


Để khắc phục tình trạng dễ dàng thay đổi của class, ta có thể sài hàm contains để thay cho so sánh tuyệt đối bằng dấu =

Vậy nên, từ nay về sau, cứ khi nào sử dụng tới class là sẽ sử dụng hàm contains, tuyệt đối ko sử dụng dấu =

//button[contains(@class , 'btn6')]
